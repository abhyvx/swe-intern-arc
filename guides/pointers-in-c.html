Pointers in C

This is the core of CMSC 216. Goal: you can explain * and & and why swap needs pointers.

Variables live at addresses

Think of memory as numbered lockers. int x = 42; puts 42 in some locker. That locker’s number is the address of x.

&x means “the address of x”. A pointer is a variable that stores an address.

int x = 42;
int *p = &x;   /* p holds the address of x */
printf("%d\n", *p); /* *p means "the value at that address" → 42 */
*p = 7;            /* write 7 into x's locker */
printf("%d\n", x); /* 7 */

Star in two places (this confuses everyone)

  • Declaration int *p → “p is a pointer to int”
  • Expression *p → “follow p to the value”

Why swap needs pointers

C passes copies of ints into functions. void swap(int a, int b) swaps copies; callers unchanged.

void swap(int *a, int *b) receives addresses, so it can write into the caller’s variables.

void swap(int *a, int *b) {
  int t = *a;
  *a = *b;
  *b = t;
}

Arrays decay

When you pass an array to a function, C passes a pointer to the first element. That is why functions often take int *a plus a length.

When you use this

  • Almost every CMSC 216 assignment
  • Understanding strings as char *
  • Later: linked lists, syscalls, anything with memory
  • Interview talk: “pass by pointer vs pass by value”

How to fix common bugs

  • segfault → often *p when p is garbage/NULL; initialize pointers
  • swapped wrong → you swapped the pointers’ values instead of *a/*b
  • type warnings → int * vs int mismatch; read the warning
